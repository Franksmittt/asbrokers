1. PROJECT OVERVIEW
--------------------------------------------------------------------------------

Project name:     as-brokers (AS Brokers website)
Repository:        https://github.com/Franksmittt/asbrokers.git
Framework:         Next.js 15 (App Router)
Language:          TypeScript
Styling:           Tailwind CSS
Key libraries:     Framer Motion, React Hook Form, Zod, clsx, tailwind-merge,
                   Recharts (calculator charts), Vercel AI SDK (ai, @ai-sdk/openai,
                   @ai-sdk/react), Drizzle ORM, pg, pgvector, server-only (HubSpot),
                   Resend (email); Trigger.dev v4 + Puppeteer (background PDF jobs).
                   Quiz: Zustand (state), nuqs (URL query state for shareable results).
                   Phase 3: next-sanity, sanity (v3), @portabletext/react; Studio deps:
                   is-hotkey, styled-components, hoist-non-react-statics, uuid (v11).
                   Phase 4: @next/third-parties (GA4); analytics only after cookie consent.

Purpose:
  Marketing and lead-generation website for AS Brokers CC, an independent
  authorised financial services provider in South Africa. The site is
  positioned for high-net-worth individuals (HNWIs) and retirees: home
  page funnels to Run-Out Calculator and Everest Wealth (Code 1.8);
  solutions and calculators hubs prioritise "Private Wealth & Yield" and
  "Retirement Reality"; retirement, tax, estate, and premium calculators
  are framed as institutional fiduciary diagnostics with CTAs to Everest
  and contact. Standard insurance and estate/medical remain as "Wealth
  Protection & Fiduciary Defense." Interactive calculators and clear
  paths to contact and book consultations.

What this project can do:
  - Full marketing site: home (Reality Check hero, RunOutCalculator, EverestWealthBento, Code18Advantage, RiskArchitectureCarousel, FloatingChat), solutions hub (Phase 1/2 hierarchy), calculators hub (Phase 1/2), team, about, contact.
  - Contact form → HubSpot CRM (search-patch, cumulative lead score); optional PDF email via Trigger.dev + Resend.
  - Digital Wealth Assistant (chat) with deterministic tools (estate duty, 12.8% income, Amethyst) and optional RAG (pgvector).
  - Financial Health Quiz with shareable result URLs (nuqs) and routing to relevant calculators.
  - Sanity CMS for insights; draft mode and Visual Editing; FSP disclaimer blocks in articles.
  - POPIA cookie consent; GA4 and Hotjar only after "Accept All".
  - PWA manifest; SEO (Organization/FinancialService schema, FAQPage, metadata per page).
  - Run-out and product calculators (retirement, tax, estate, inflation, Everest Wealth, Amethyst).
  - CRM & Client Portal: mock-auth login (/login); role-based CRM (dashboard, Kanban, leads, clients, tasks, lead/client detail with correspondence, notes, reassign); Client Portal (wealth dashboard, My Advisor, documents with upload, messages with reply). See CRM_AND_PORTAL.txt.

Design (current layout & styling):
  Fused "Apple Pro" and "Samsung Galaxy" design system for an ultra-luxury,
  futuristic private wealth terminal look.

  Colour & lighting:
    - Backgrounds: absolute black (--bg-primary: #000000, Tailwind bg-void) or Shark Grey (#1D1D1F, bg-shark).
    - Rim-light glassmorphism: .rim-light in globals.css (bg-white/5 backdrop-blur-2xl ring-1 ring-white/10 + inset edge highlight). Used for nav (when scrolled), cards, dropdowns, CTAs.
    - Nightography ambient: BackgroundOrbs — large, heavily blurred absolute divs with Cinematic Teal (#008080) and Gold/Orange (#FF7F50) for OLED contrast; mouse-reactive with Apple easing.

  Geometry & layout:
    - Bento grid masonry for solutions and calculator hub cards.
    - Squircle: rounded-[2rem] or rounded-3xl (Tailwind) for organic curvature; no sharp corners on cards.
    - Generous padding and whitespace within cards (luxury signal).

  Typography:
    - Inter as primary font (--font-inter). Headlines: large, bold, tight tracking (tracking-[-0.03em]). Body: text-gray-400, slightly looser tracking (tracking-[0.01em]). Trust signals use .trust-hallmark (tabular-nums, letter-spacing, engraved look).

  Motion (Framer Motion):
    - Apple-style easing: cubic-bezier(0.25, 0.1, 0.25, 1) (ease-apple in Tailwind); duration 0.5–0.8s for page loads and layout changes.
    - CTA hovers: Samsung Blue (#0057B8) or Supernova Gold glow (shadow-cta-glow-blue / shadow-cta-glow-gold) with subtle scale-up (hover:scale-[1.03]).

  Trust signals:
    - FSP 17273, 25+ years, Code 1.8 rendered with .trust-hallmark for architectural precision (TrustBar, HomeStatsSection, Nav).

  Tokens (tailwind.config.ts & globals.css):
    - void, shark, cinematic-teal, gold-orange, samsung-blue, supernova-gold; transitionTimingFunction.apple; boxShadow rim-glow, cta-glow-blue, cta-glow-gold. Legacy .glass-card retained for compatibility.

10. RAG PIPELINE (RETRIEVAL-AUGMENTED GENERATION)
--------------------------------------------------------------------------------

Purpose: Ground the chat assistant in Everest Wealth and FAIS/SARS documentation
via pgvector similarity search. Baseline constraints remain in code; RAG adds
retrieved context only.

Database (Drizzle ORM + PostgreSQL + pgvector):
  - lib/db/schema.ts:
    - resources: id, content (text), source (varchar 255), metadata (jsonb), created_at
    - embeddings: id, resource_id (FK → resources.id ON DELETE CASCADE), embedding (vector(1536)), created_at
    - HNSW index on embeddings.embedding with vector_cosine_ops for cosine similarity
  - Extension: CREATE EXTENSION vector; (run manually or via custom migration)
  - drizzle.config.ts: schema ./lib/db/schema.ts, out ./drizzle, dialect postgresql, dbCredentials.url from DATABASE_URL

Retrieval:
  - lib/db/rag.ts:
    - getRagContext(query, limit?) → Promise<string>
    - Embeds query via AI SDK embed + openai.embedding('text-embedding-3-small')
    - If DATABASE_URL missing: returns "" (no DB connection)
    - Otherwise: Drizzle select from embeddings join resources, orderBy sql`embedding <=> $vector::vector`, limit 5
    - Returns concatenated "[Source: ...]\ncontent" blocks or "" on error
  - lib/db/index.ts: getDb() returns Drizzle client or null; lazy pool from DATABASE_URL

Chat route (app/api/chat/route.ts):
  - getLatestUserMessageText(messages) → last user message text from parts/content
  - systemPrompt = BASELINE_SYSTEM_PROMPT + "\n\n[Retrieved context]:\n" + (await getRagContext(latestText, 5)) or "[No relevant context retrieved.]"
  - streamText({ system: systemPrompt, ... }) — baseline constraints untouched

Environment:
  - OPENAI_API_KEY: required for chat (and for RAG embedding)
  - DATABASE_URL: optional; if set, RAG retrieval runs; if unset, chat still works with empty retrieved context

Document ingestion:
  - scripts/ingest-rag.ts: chunks .txt/.md from scripts/rag-documents (or custom dir), embeds with text-embedding-3-small, inserts into resources + embeddings. Run: npm run rag:ingest (needs OPENAI_API_KEY, DATABASE_URL). Plan: docs/RAG_IMPLEMENTATION_PLAN.md.

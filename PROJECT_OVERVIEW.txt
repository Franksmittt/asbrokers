1. PROJECT OVERVIEW
--------------------------------------------------------------------------------

Project name:     as-brokers (AS Brokers website)
Repository:        https://github.com/Franksmittt/asbrokers.git
Framework:         Next.js 15 (App Router)
Language:          TypeScript
Styling:           Tailwind CSS
Key libraries:     Framer Motion, React Hook Form, Zod, clsx, tailwind-merge,
                   Recharts (calculator charts), Vercel AI SDK (ai, @ai-sdk/openai,
                   @ai-sdk/react), Drizzle ORM, pg, pgvector, server-only (HubSpot),
                   Resend (email); Trigger.dev v3 + Puppeteer (dev, background PDF jobs).
                   Quiz: Zustand (state), nuqs (URL query state for shareable results).
                   Phase 3: next-sanity, sanity (v3), @portabletext/react; Studio deps:
                   is-hotkey, styled-components, hoist-non-react-statics, uuid (v11).
                   Phase 4: @next/third-parties (GA4); analytics only after cookie consent.

Purpose:
  Marketing and lead-generation website for AS Brokers CC, an independent
  authorised financial services provider in South Africa. The site presents
  retirement planning, insurance, estate planning, and Everest Wealth
  fixed-return products, with interactive calculators and clear paths to
  contact and book consultations.

Latest repo update:
  March 2026: Phase 4 Analytics & POPIA. GA4 (@next/third-parties) and Hotjar
  (async script) load only after "Accept All" in cookie consent banner. Consent
  stored in localStorage; banner (glassmorphism, POPIA copy) fixed bottom with
  Accept All / Essential Only. Phase 3: Sanity v3 CMS, insights, draft mode,
  Visual Editing. Phase 2: Trigger.dev PDF, quiz (Zustand + nuqs). HubSpot, RAG, chatbot, SEO.

Data currency (2026):
  Income tax and Amethyst living-annuity tax estimates use SARS 2026/27
  brackets and rebates (Budget 2026). Estate duty, donations (R100k/R200k),
  and other calcs use current legislation; review periodically.

Design:
  Dark theme (#0a0a0c, #151518), glassmorphism nav on scroll, gradient orbs
  on hero (parallax/mouse-reactive via Framer Motion). Primary cards use
  glass-card: backdrop-blur-xl, ring/border, edge illumination (see globals.css).
  Responsive (mobile, tablet, desktop) with frosted header (80% opacity) when
  scrolling. PWA manifest for installability. Accessibility: skip link to main
  content, prefers-reduced-motion respected. See ROADMAP.md for phases.

--------------------------------------------------------------------------------
10. RAG PIPELINE (RETRIEVAL-AUGMENTED GENERATION)
--------------------------------------------------------------------------------

Purpose: Ground the chat assistant in Everest Wealth and FAIS/SARS documentation
via pgvector similarity search. Baseline constraints remain in code; RAG adds
retrieved context only.

Database (Drizzle ORM + PostgreSQL + pgvector):
  - lib/db/schema.ts:
    - resources: id, content (text), source (varchar 255), metadata (jsonb), created_at
    - embeddings: id, resource_id (FK → resources.id ON DELETE CASCADE), embedding (vector(1536)), created_at
    - HNSW index on embeddings.embedding with vector_cosine_ops for cosine similarity
  - Extension: CREATE EXTENSION vector; (run manually or via custom migration)
  - drizzle.config.ts: schema ./lib/db/schema.ts, out ./drizzle, dialect postgresql, dbCredentials.url from DATABASE_URL

Retrieval:
  - lib/db/rag.ts:
    - getRagContext(query, limit?) → Promise<string>
    - Embeds query via AI SDK embed + openai.embedding('text-embedding-3-small')
    - If DATABASE_URL missing: returns "" (no DB connection)
    - Otherwise: Drizzle select from embeddings join resources, orderBy sql`embedding <=> $vector::vector`, limit 5
    - Returns concatenated "[Source: ...]\ncontent" blocks or "" on error
  - lib/db/index.ts: getDb() returns Drizzle client or null; lazy pool from DATABASE_URL

Chat route (app/api/chat/route.ts):
  - getLatestUserMessageText(messages) → last user message text from parts/content

Purpose: Enterprise analytics (GA4, Hotjar) with strict POPIA compliance: no

Cookie consent (components/ui/CookieConsent.tsx):

  - Buttons: "Accept All" (enables GA4 + Hotjar), "Essential Only" (no analytics).

  - React context: consent ("all" | "essential" | null), setConsent, hasChosen.
  - On mount, reads localStorage; ConsentProvider wraps app and renders
    CookieConsent + ConditionalAnalytics.

  - GA4: @next/third-parties/google <GoogleAnalytics gaId="..." />. Mounted only
    when consent === "all" (ConditionalAnalytics).
  - Hotjar: components/analytics/HotjarAnalytics.tsx; next/script strategy="lazyOnload",
    injected only when consent === "all". Snippet loads Hotjar async.
  - If user has not consented or chooses "Essential Only", GA4 and Hotjar never mount.

Root layout:
  - ConsentProvider wraps MotionConfigProvider and app content; renders
    CookieConsent (when hasHydrated) and ConditionalAnalytics.

Env:
  - NEXT_PUBLIC_GA_MEASUREMENT_ID (GA4 measurement ID, e.g. G-XXXXXXXX).
